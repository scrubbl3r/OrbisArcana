What to update when adding real local assets

1. Set SDK version
   - `PORCUPINE_LOCAL_SDK_VERSION`

2. Confirm vendor SDK runtime file names/paths
   - `scriptPath`
   - `wasmPath`
   - `workerPath`

3. Drop keyword model files and update model path mapping if filenames differ
   - `PORCUPINE_LOCAL_KEYWORD_MODEL_PATHS`

Current expected layout (default)

- /vendor/porcupine/<version>/porcupine.js
- /vendor/porcupine/<version>/porcupine.wasm
- /vendor/porcupine/<version>/porcupine-worker.js
- /assets/kws/keywords/<token>.ppn

Notes

- Receiver KWS backend availability depends on this config being non-placeholder.
- Keep this file as the single path/version source instead of hardcoding paths in
  the receiver or provider modules.
