KWS LOCAL BUNDLE BOOT FLOW (v1)

Goal

Enable local Porcupine-style KWS assets without changing receiver code each time.

Current boot chain (implemented)

Receiver startup:
  game-receiver.js
    -> imports src/voice/kws/porcupine-local-boot.js
    -> calls bootLocalPorcupineKws()
    -> best-effort install of window.OrbisKwsBackendFactory

How local boot works

1. Reads local manifest from `porcupine-init.js`
2. Tries to load local SDK script (if manifest is configured)
3. Looks for `window.OrbisCreatePorcupineSdkSession`
4. Installs `window.OrbisKwsBackendFactory` via the adapter
5. Receiver KWS provider uses that backend when KWS Mic is toggled on

What you need to add for real KWS

1. Vendor SDK assets (pinned)
   - vendor/porcupine/<version>/...

2. Keyword models
   - assets/kws/keywords/*.ppn

3. SDK session wrapper on window
   - `window.OrbisCreatePorcupineSdkSession = async ({ stream, keywords, onDetection, onError }) => ...`
   - skeleton file provided in `porcupine-sdk-session-global.js`

4. Update manifest in `src/voice/kws/porcupine-init.js`
   - set real version
   - set script/wasm/worker paths
   - set real keyword model paths

Expected receiver UI behavior

Before SDK is wired:
- KWS readout shows `backend:none`
- KWS Mic button disabled

After SDK is wired correctly:
- KWS readout shows `backend:ready`
- KWS Mic button enabled
- KWS Shadow + Mic On shows token/candidate events in HUD log
